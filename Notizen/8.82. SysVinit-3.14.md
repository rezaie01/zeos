#sysvinit

### Funktion des Pakets

- Steuerung von Systemstart, Laufbetrieb und Herunterfahren
- Implementierung klassischer SysV-Init-Utilities und Runlevel-Management

### Kompilierungsprozess (exakt wie im Buch)

```bash
patch -Np1 -i ../sysvinit-3.14-consolidated-1.patch
```

```bash
make
```

```bash
make install
```

### Nach-der-Installation-Konfiguration

- Keine zusätzliche Konfiguration im LFS-Abschnitt angegeben
- Init liest Konfiguration/Startskripte gemäß lokalem SysV-Layout (/etc/inittab, /etc/rc.d o.ä.), falls vorhanden

### Essenzielle Programme und Nutzung (Analyse: Systemintegration → alle 10/10, auswendig lernen)

- #init
    
    - Wichtigkeit: 10/10
    - Zweck: PID 1; übernimmt Boot-Prozess, startet Prozesse gemäß Konfiguration; zentral für Systemzustand
    - Typische Nutzung / Verhalten (merkfähig): startet beim Kernel-Start automatisch; steuert Runlevel-Wechsel über telinit oder inittab-Einträge
- #telinit
    
    - Wichtigkeit: 10/10
    - Zweck: Teilt init mit, auf welchen Runlevel gewechselt werden soll
    - Beispiele (merken):
        - telinit 0 — Wechsel zu Runlevel 0 (Herunterfahren/Halt)
        - telinit 6 — Wechsel zu Runlevel 6 (Reboot)
        - telinit 1 — Wechsel zu Einzelbenutzer-/Wartungsmodus
- #shutdown
    
    - Wichtigkeit: 10/10
    - Zweck: Sicheres Herunterfahren; signalisiert Prozesse, benachrichtigt Nutzer, plant Systemhalt oder Reboot
    - Übliche Verwendung (merken):
        - shutdown -h now — sofortiges Herunterfahren und Halt
        - shutdown -r +5 — Neustart in 5 Minuten
- #halt / #poweroff / #reboot
    
    - Wichtigkeit: 10/10
    - Zweck: Direktes Veranlassen von Herunterfahren/Netzaus/Neustart; halt ruft im Runlevel 0 Kernel-Halt, poweroff schaltet aus, reboot startet neu
    - Merkhinweis: poweroff und reboot sind Links auf halt; Verhalten kann kontextabhängig (Runlevel) sein
- #runlevel
    
    - Wichtigkeit: 10/10
    - Zweck: Meldet vorherigen und aktuellen Runlevel (laut /run/utmp)
    - Nutzung (merken): runlevel — zeigt Wechselinfo; nützlich nach telinit/init-Aktionen zur Verifikation
- #killall5
    
    - Wichtigkeit: 10/10
    - Zweck: Sendet ein Signal an alle Prozesse außer der eigenen Session; nützlich beim Runlevel-Wechsel/Shutdown
    - Nutzung (merken): killall5 SIGTERM — breit Signal an Prozesse senden (Vorsicht: global)
- #bootlogd
    
    - Wichtigkeit: 7/10
    - Zweck: Protokolliert Boot-Messages in eine Log-Datei; nützlich zur Diagnose von Bootproblemen
    - Nutzung: Bei Bedarf aktivieren/überprüfen, wo Boot-Logs abgelegt werden (z. B. /var/log)

---

### Wichtige Pfade / Hinweise

- /var/log/wtmp wird bei halt/reboot/boot-events aktualisiert (runlevel / login-history)
- Init-Konfiguration/Startskripte können distributions- oder system-spezifisch sein (/etc/inittab, /etc/rc.d)
- SysVinit ersetzt nicht systemd; Integrationspunkt: Bootprozess, Runlevel-Management, sauberer Shutdown

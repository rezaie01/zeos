## 9.8. Erstellen der `/etc/inputrc` Datei

Die Datei `/etc/inputrc` ist die Konfigurationsdatei für die `readline`-Bibliothek. Die `readline`-Bibliothek ist dafür da, Bearbeitungsfunktionen zu ermöglichen, während ich Text in das Terminal eingebe. Sie macht das, indem sie meine Tastatureingaben in Aktionen übersetzt. Shells wie Bash und viele andere Anwendungen nutzen `Readline`.

Ich erstelle die `/etc/inputrc`, weil die meisten Leute keine benutzerspezifischen Einstellungen brauchen und so alle, die sich anmelden, die gleichen globalen Einstellungen nutzen können. Wenn ich später doch mal individuelle Anpassungen brauche, kann ich einfach eine `.inputrc` in meinem Home-Verzeichnis erstellen, die dann die globalen Mappings überschreibt. Für mehr Infos kann ich `info bash` im Abschnitt *Readline Init File* oder `info readline` lesen.

### Erklärung des Codeblocks:

Dieser Block erstellt die Konfigurationsdatei für die `readline`-Bibliothek.

```
CODEBLOCK 1:
cat > /etc/inputrc << "EOF" # Begin /etc/inputrc # Modified by Chris Lynn <roryo@roryo.dynup.net>
# Allow the command prompt to wrap to the next line set horizontal-scroll-mode Off
# Enable 8-bit input set meta-flag On set input-meta On
# Turns off 8th bit stripping set convert-meta Off
# Keep the 8th bit for display set output-meta On
# none, visible or audible set bell-style none
# All of the following map the escape sequence of the value # contained in the 1st argument to the readline specific functions "\eOd": backward-word "\eOc": forward-word
# for linux console "\e[1~": beginning-of-line "\e[4~": end-of-line "\e[5~": beginning-of-history "\e[6~": end-of-history "\e[3~": delete-char "\e[2~": quoted-insert
# for xterm "\eOH": beginning-of-line "\eOF": end-of-line
# for Konsole "\e[H": beginning-of-line "\e[F": end-of-line
# End /etc/inputrc EOF
```

Die Zeilen setzen die verschiedenen Modi (`set horizontal-scroll-mode Off`, `set meta-flag On`, etc.) zur Steuerung des Verhaltens. Die Zuordnungen (Mappings) wie `"\eOd": backward-word` sind wichtig, damit Funktionstasten und Steuerungstasten in verschiedenen Terminalumgebungen (Linux Konsole, Xterm, Konsole) korrekt erkannt und in die entsprechenden `readline`-Aktionen übersetzt werden.

## Glossar

| Begriff | Erklärung (LFS-Kontext) |
| :--- | :--- |
| **Login Shell** | Eine Shell, die nach einer erfolgreichen Anmeldung (Login) gestartet wird, typischerweise über `/bin/login`. |
| **Non-Login Shell** | Eine interaktive Shell, die direkt auf der Kommandozeile gestartet wird. |
| **Non-Interactive Shell** | Eine Shell, die ein Skript verarbeitet und keine Benutzereingaben zwischen den Befehlen erwartet. |
| **Locale** | Ein Satz von Umgebungsvariablen, die länderspezifische Einstellungen definieren (Sprache, Zeichensatz, Währung, etc.). |
| **C Locale (C)** | Die Standard-Locale, die das 7-Bit US-ASCII Zeichenset verwendet und 8-Bit-Zeichen als ungültig behandelt. |
| **C.UTF-8 Locale** | Eine Locale, die UTF-8 unterstützt. Im LFS-Kontext wird sie in der Linux-Konsole verwendet, um Anzeigefehler zu vermeiden. |
| **Readline Library** | Eine Bibliothek, die Bearbeitungsfunktionen bereitstellt, während der Benutzer Eingaben über das Terminal macht (wird von Bash und anderen Anwendungen genutzt). |
| **inputrc** | Die Konfigurationsdatei für die Readline Library, die Tastatureingaben in Aktionen übersetzt. |
| **$TERM** | Eine Umgebungsvariable, die den Typ des verwendeten Terminals angibt (z. B. `linux` für die Konsolendarstellung). |
